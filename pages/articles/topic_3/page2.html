Topic # 2

Knowing the Syntax Is Not the Same as Knowing How to Code

LEONGSUN, EUWIN March 31, 2026 5 min read

There is a big difference between memorizing how a for loop works and actually knowing how to use it to solve a real problem. I learned that the hard way when I first tried building something outside of school and realized my class notes were not going to save me.

You can pass every programming subject your course throws at you and still struggle to build a working application from scratch. That sounds harsh but it is just the reality of how most CS and IT programs are structured. They teach you the theory. They teach you the language. What they do not always teach you is how to actually use all of that in a real setting.

What School Focuses On vs. What You Actually Need

School is good at teaching you concepts. Data structures, algorithms, object-oriented programming, system design theory. These matter, and you will need them. But when you sit down to actually build something, you run into a completely different set of problems.

How do you use version control so you do not lose your work or break everything when you are collaborating? How do you connect your application to a database and actually query it? How do you call an external API and handle the response? How do you put your app somewhere people can access it? These are practical skills, and most of them you have to figure out on your own.

That does not mean school is useless. The foundation matters. But it does mean you cannot just rely on class to make you competent. You have to seek out the practical side yourself.

Technical Skills Worth Building Right Now

If you are in a tech-related program and want to close the gap between what you know and what you can actually do, these are the areas worth putting time into.

Start with Git. Version control is non-negotiable in any real development environment and it is something most students avoid because it feels intimidating at first. Learn the basics, start using it on your own projects, and it will become second nature fast.

Learn how to work with APIs. Most modern applications pull data from somewhere, and knowing how to make an HTTP request, handle a response, and do something useful with the data is a skill that shows up constantly. Pick any free public API and try building something with it.

Get comfortable with databases. You do not need to master everything. Just understand how to set one up, write basic queries, and connect it to an application. SQLite or MySQL is fine to start with.

Try deploying something. Even if it is just a simple static site. The process of taking code off your machine and putting it somewhere the world can access it teaches you things you cannot learn from a textbook. You will run into environment problems, configuration issues, things that worked locally but break in production. All of that is valuable.

Last one: learn to read error messages properly. This sounds obvious but most beginners either panic or just Google the whole error without actually reading it. The error message usually tells you exactly where things went wrong. Training yourself to slow down and read it carefully saves an enormous amount of time.
